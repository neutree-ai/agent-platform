---
title: 2. 第一个 Agent
description: 从零创建一个 Workspace 并完成第一次对话
---

[指南 1](/guides/1-setup/) 里说的 API 供应商已经就绪后，可以创建你的第一个 Agent 了。整个过程不超过 5 分钟。

## 创建 Workspace

点击侧边栏的 **创建工作空间**，在弹窗里：

1. 填 **名称** ——比如 `my-first-agent`
2. 选择 **模式**：
   - **从模板创建** ——从预设模板创建。模板已经包含完整的 Agent 配置（API 供应商、模型、Prompt，可能还有 skills），适合刚开始
   - **空白** ——从零选择 API 供应商、模型、Prompt。适合你已经清楚要怎么配的情况

第一次强烈推荐 **从模板创建**。如果平台里有名为 `quick-start` 的模板，直接选它——这是为新用户准备的最小可用配置。

如果没有任何模板，那就选 **空白**，然后：

- **API 供应商** ——选择上一章准备好的那个
- **模型** ——选一个具体的模型（比如 `gpt-5.4`）
- **Prompt** ——可以先留空，进入 Workspace 后再写

点击 **创建**。

## 进入 Workspace

Workspace 创建后会自动启动并打开。第一次启动需要等几秒——Neutree Agent Platform 在后台为你的 Agent 准备运行实例。

启动完成后你会看到几个区域：

| 区域 | 用途 |
|---|---|
| 顶部 tab 栏 | **文件 / 终端** 是 Agent 的运行环境；**Agent 配置 / 自动化 / Memory** 是配置入口 |
| 中间主区 | 当前打开的 tab 内容 |
| 右侧对话面板 | 和 Agent 对话的地方 |
| 顶部会话栏 | 切换或新建会话 |

打开任意配置项时，弹窗右侧都会显示对应的字段说明。遇到不确定的地方直接看右侧即可，不需要切到这份文档。

## 第一次对话

如果你用了 `quick-start` 模板，prompt 已经就位，直接在右下角输入框发送一条消息：

```
你好，请用一句话介绍你自己
```

Agent 会回复。你可以接着追问、发图片、贴链接，或者让它做一些简单的事：

```
帮我列出当前工作目录下的所有文件
```

Agent 会调用终端命令查看，并把结果回复给你。

如果 prompt 还没写，Agent 会以通用助手的身份回应——能聊但没有特定的工作风格。等你完成 prompt 之后再来对话，你会看到完全不同的 Agent。

## 看一眼 文件 和 终端

切到顶部的 **文件** tab，你会看到 Agent 工作目录下的所有文件——刚才让它"列文件"时它看到的就是这些。

切到 **终端** tab，可以直接进入 Agent 容器的 shell。这里执行的命令和 Agent 自己执行的命令在同一个环境里——你写一个文件，Agent 下次就能读到。

文件和终端是**跨 Session 共享**的：你新建一个 Session 时，文件还在原处。

## 接下来

到这里你有了一个能跑、能对话的 Agent。下一步是让它做你真正想让它做的事——写 prompt 让它有"人格"和做事方式、加 skills 让它具备特长、配 MCP 让它接外部系统、用 memory 让它越用越懂你。这些都在 [指南 3：定义 Agent 行为](/guides/3-agent-behavior/)。
