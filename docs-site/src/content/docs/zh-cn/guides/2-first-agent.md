---
title: 2. 第一个 Agent
description: 从零创建一个 Workspace 并完成第一次对话
---

[准备工作](/zh-cn/guides/1-setup/)里的 API 供应商就绪后，可以创建你的第一个 Agent 了。整个过程不超过 5 分钟。

## 创建 Workspace

点击顶部菜单栏的 workspace 切换器，选择 **创建工作空间**，在弹窗里：

1. 填 **名称** —— 比如 `my-first-agent`
2. 选择 **模式**：
   - **从模板创建** —— 从实例里已共享的模板创建。模板包含完整的 Agent 配置（供应商、模型、Prompt，可能还有 skills）；有合适的直接选，这一步就完成了
   - **空白** —— 从零配置。全新安装还没有模板时走这条路 —— 其实只有几个字段：
     - **Agent Type** —— Claude Code 或 Codex
     - **API 供应商** —— 列表只显示与所选 agent type 兼容的供应商（对应关系见[准备工作](/zh-cn/guides/1-setup/)）
     - **模型** —— 选一个该供应商实际提供的模型
     - **Prompt** —— 可以先留空，进入 Workspace 后再写

点击 **创建**。

## 进入 Workspace

Workspace 创建后会自动启动并打开。第一次启动需要等几秒 —— 平台在后台为你的 Agent 准备运行实例。

默认布局是三列：

| 区域 | 用途 |
|---|---|
| 左列 —— **会话历史** | 切换或新建会话 |
| 中列 —— 工作区 | App tab：**文件 / 浏览器 / Skills / 终端 / 自动化 / 记忆库 / 设置** —— Agent 的环境和配置都在这里 |
| 右列 —— **对话** | 和 Agent 对话的地方 |

布局可以随意调整 —— app 可以在列之间移动、多开、或弹出成独立窗口；`⌘K` 随时直达任何地方。

打开任意配置项时，弹窗右侧都会显示对应的字段说明。遇到不确定的地方直接看右侧即可，不需要切回这份文档。

## 第一次对话

在右侧对话输入框发送一条消息：

```
你好，请用一句话介绍你自己
```

Agent 会回复。你可以接着追问、发图片、贴链接，或者让它做一些简单的事：

```
帮我列出当前工作目录下的所有文件
```

Agent 会在自己的环境里执行命令，并把结果回复给你。

如果 prompt 还没写，Agent 会以通用助手的身份回应 —— 能聊但没有特定的工作风格。等你完成 prompt 之后再来对话，你会看到完全不同的 Agent。

## 看一眼文件

打开中列的 **文件** tab，你会看到 Agent 的工作目录 —— 刚才让它「列文件」时它看到的就是这些。你放进来的东西 Agent 都能读到；它产出的文件也会出现在这里。

文件是**跨 Session 共享**的：新建一个 Session 时，文件还在原处。

## 接下来

到这里你有了一个能跑、能对话的 Agent。下一步是让它做你真正想让它做的事 —— 写 prompt 让它有「人格」和做事方式、加 skills 让它具备特长、配 MCP 让它接外部系统、用 memory 让它越用越懂你。这些都在[定义 Agent 行为](/zh-cn/guides/3-agent-behavior/)。
