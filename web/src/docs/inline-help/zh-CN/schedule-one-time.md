一次性定时任务在指定时间点触发一次，触发后自动标记为 **已完成**，不会再次触发，也不需要手动清理。

适用于"3 天后帮我整理一次报表"、"下周一发布前跑一遍 QA"这类**单次**需求；不需要构造 cron 表达式，也不需要在执行后手动删除任务。

## 字段

- **Name** — 任务名称，用于列表展示和日志标识
- **执行时间** — 触发的具体时间点，必须是未来时间
- **Timezone** — 仅作展示标记；触发的精确时刻已由所选时间确定
- **Prompt** — 触发时发送给 agent 的指令内容；可直接填写文本，也可引用 prompt 库中的模板

## 注意事项

- 已完成的一次性任务保留在列表中作为历史记录，不可再启用或编辑；如需重做，直接新建一个即可
- **禁用**一个未触发的一次性任务相当于取消该次触发；重新启用后会按原时间继续等待，过去的时间不会被补触发
- 在 agent 对话里也可以用自然语言让 agent 帮你创建、修改、删除一次性任务
